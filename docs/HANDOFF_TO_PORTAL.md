## Critical: RLS Infinite Recursion Fix

**Problem found and fixed:** All admin policies (`centers_admin_all`, `donations_admin_all`, etc.) used inline `user_roles` subqueries which caused infinite recursion for ALL users (including anon), breaking every Supabase read on the public site.

**Fix applied (migration: `fix_rls_infinite_recursion_is_admin_function`):**
```sql
CREATE OR REPLACE FUNCTION public.is_admin()
RETURNS BOOLEAN LANGUAGE SQL SECURITY DEFINER SET search_path = public STABLE AS $$
  SELECT EXISTS (SELECT 1 FROM public.user_roles WHERE user_id = auth.uid() AND role = 'admin')
$$;
```
All 7 admin policies now call `is_admin()` instead of inline subqueries.

**If you create new admin policies on any table, use `is_admin()` not inline subqueries.**
